# Course Topic List


| Topic and Content Covered in the Course | Translation and Professional Context | The Revolution and Value for You (The Value) |
| --- | --- | --- |
| Machine Learning Algorithms - covered superficially because there are topics important to know | Machine Learning Algorithms: the mathematical and statistical foundation (regressions, trees, etc.). | From superficial understanding to structural understanding: understanding the physical limitations of the model. Value: You'll know when to use an LLM and when a simple algorithm will solve the problem at a fraction of the cost. |
| Generative AI - covered more in depth as this is the course focus | Generative Artificial Intelligence (GenAI): models that generate new content (text, code, image). | The computational creativity revolution: understanding the Transformer architecture. Value: Reducing "hallucinations" through a deep understanding of how the mechanism works. |
| Python for AI Development - we'll learn Python while learning only what's relevant to the field | Python for AI Development: using libraries such as: Pandas, NumPy and the SDK libraries of language models. | From writing code to building data engines: advanced use of data processing libraries. Value: The ability to stream clean and accurate information to models - the foundation of every AI system. |
| Using AI coding assistants - we'll work with Cursor or Copilot writing assistants | Efficient use of AI programming assistants: working with tools such as: Copilot or Cursor to improve development speed. | From writing code to Code Review: changing your teams' work paradigm. Value: Managing Velocity while preventing "technical debt" and code that developers don't fully understand. |
| MLOps DevOps - connection between the software world and the Operation world, using containers | ML Lifecycle Management: automation of deployment, monitoring and updating models (similar to DevOps). | From experiment (PoC) to production system: the hardest transition in the AI world. Value: Building an automated Pipeline that deploys and monitors models without the system crashing under load. |
| Frontier vs Open Source models - we'll learn about OpenAI, Claude, Gemini models and more, there will also be work with Open Source tools | Closed (Frontier) vs. Open Source models: comparison between GPT/Gemini and models like Llama. | From vendor dependency to strategic independence: understanding when to "rent" a model (OpenAI) and when to "build" (Llama). Value: Protecting customer privacy and saving costs by running custom local models. |
| RAG (Retrieval Augmentation Generation) - we'll learn how I improve my query results against data I choose | Retrieval Augmented Generation (RAG): "connecting" the model to organizational data to prevent hallucinations. | From general information to specific organizational information: preventing model "hallucinations" based on facts. Value: Architectural challenge of retrieving the most accurate information from huge databases to produce truthful answers. |
| MCP (Model Context Protocol) - we'll learn how to enable AI to talk to external systems | Model Context Protocol: a new standard (from Anthropic) that allows models uniform access to data. | The quiet revolution in data connectivity: standardization of Context. Value: Connecting the model to APIs and your data securely without writing "glue code" for each model. |
| AI Agents & Agentic Design - we'll learn how to build AI Agents properly and efficiently, teaching how to connect them to the real world | AI Agents and Agentic Design Patterns: transitioning from a model that "answers" to a model that "acts" (plans and executes tasks). | From the stage of "asking questions" to the stage of "executing tasks": Value: Building an autonomous system that acts (for example: an agent that finds a fault in the call center, opens a Jira ticket and updates the customer). |
| LangChain, LangGraph, CrewAI - we'll discuss libraries that help us build AI Agents, we'll also discuss how to build them in Pure Python | AI Frameworks: Python libraries for building LLM applications and State management. | From simple conversation to complex application: using the right tools for logic management. Value: Ability to manage complex Workflows and maintain "State" over time. |
| Multi-Agent Orchestration - we'll learn how to work with Multi-Agent and understand why we need them | Multi-Agent System Orchestration: managing a team of AI agents working together (one programs, one tests). | Virtual team management: dividing a large task into focused subtasks. Value: Orchestrating different agents (one for transcription, one for analytics) to create a quality and controlled final product. |
| Performance and Scalability - we'll also cover this topic | Performance and Scalability: how to make an AI system work fast and serve thousands of users. | From slow system to Enterprise system: handling AI bottlenecks. Value: You'll learn how to manage response times (Latency) and token costs in systems serving thousands of users simultaneously. |
| Advanced AI Engineering - managing Context is one of the most important things in the industry | Advanced AI Engineering: specialization in Context Engineering (building the perfect context for the model). | The evolution of Prompt Engineering: full control over the model's Input. Value: Delivering the most relevant information to achieve maximum results at minimum cost. |
| Monitoring and Logging - we'll also cover this topic | Monitoring and Logging: tracking the "health" of the model, costs (Tokens) and answer accuracy. | From guesses to diagnostics: the ability to reproduce "why did the machine answer like that?". Value: Building a transparent system that enables Debugging and control over costs and quality of products in real-time. |
| Real life AI applications - we'll develop applications that can be used in the real world and this will also be the main project of the course, we'll actually build an AI Agent that is a Customer support Agent | Real-World Applications: case studies of AI implementation in existing products. | The bottom line: how to turn all this technology into business value (ROI) in products like those you're developing. Value: Applying all technical knowledge for a final product that solves real problems for users. |
