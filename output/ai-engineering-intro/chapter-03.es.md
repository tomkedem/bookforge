# Ejemplos de Características de ML (o Features de ML)


| Característica | Explicación Técnica/Empresarial |
| --- | --- |
| Speech to text | Conversión de ondas de sonido a texto escrito en tiempo real o retroactivamente. |
| Semantic Categories | Clasificación de oraciones por significado (temas) y no solo por palabras clave (por ejemplo: reconocer que "quiero irme", "quiero terminar el contrato" pertenece a la categoría "soporte técnico"). |
| Sentiment Analysis | Identificación del estado emocional del cliente (positivo, negativo, neutral) basado en el tono de voz y la selección de palabras. |
| Call Summaries | Uso de GenAI para crear un resumen automático de la llamada, ahorrando al agente la necesidad de escribir un resumen manual. |
| Prompt Customization | La capacidad de definir instrucciones personalizadas para modelos de lenguaje (LLMs) para obtener resultados más precisos para la organización. |
| Automatic Quality Management | Control automático sobre todas las llamadas (en lugar de escucha muestral) para garantizar el cumplimiento de procedimientos y calidad del servicio. |
| AI Insights - Automatic Call Analysis | Extracción de conclusiones profundas de todas las interacciones sin intervención humana. |
| Discovery - Intent / Resolution / Clustering | Identificar qué quería el usuario (Intent), si el problema fue resuelto (Resolution), agrupar llamadas similares en grupos (Clustering) para identificar nuevas tendencias, entender qué hay en las llamadas. |



**Puntos para Considerar:**

• **Discovery (Clustering):**

Una herramienta particularmente poderosa. En lugar de decirle al sistema qué buscar, el sistema usa Unsupervised Learning para decirte: "Presta atención, de repente tienes 200 llamadas sobre un problema que no conocías antes."

• **Prompt Customization:**

Hoy, una parte significativa de la Lógica de estos productos pasa de estar codificada (hard-coded) a gestionar prompts complejos contra modelos como GPT o Gemini.

• **Intent vs. Resolution:**

La capacidad de distinguir entre lo que el cliente quería y si recibió una respuesta es la medida principal del ROI de tales sistemas.



En el mundo del Machine Learning, la palabra **Features** tiene un doble significado, y ambas interpretaciones son relevantes para la lista que presentamos en la tabla:

1. **Características (en el sentido del producto - Product Features):**

Capacidades funcionales que el sistema ofrece al usuario final gracias a la inteligencia artificial (como: transcripción o resúmenes de llamadas). Este es el significado más probable del título de la lista.

2. **Características (en el sentido técnico - Model Features):**

Las variables o datos utilizados como entrada al modelo matemático (como: duración de la llamada, palabras específicas dichas, o tono de voz) para producir la predicción.
