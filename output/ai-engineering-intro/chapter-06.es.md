# Lista de Temas del Curso


| Tema y Contenido Cubierto en el Curso | Traducción y Contexto Profesional | La Revolución y el Valor para Ti (El Valor) |
| --- | --- | --- |
| Machine Learning Algorithms - cubierto superficialmente porque hay temas importantes que conocer | Algoritmos de Machine Learning: la base matemática y estadística (regresiones, árboles, etc.). | De comprensión superficial a comprensión estructural: entender las limitaciones físicas del modelo. Valor: Sabrás cuándo usar un LLM y cuándo un algoritmo simple resolverá el problema a una fracción del costo. |
| Generative AI - cubierto más en profundidad ya que es el enfoque del curso | Inteligencia Artificial Generativa (GenAI): modelos que generan nuevo contenido (texto, código, imagen). | La revolución de la creatividad computacional: entender la arquitectura Transformer. Valor: Reducir "alucinaciones" a través de una comprensión profunda de cómo funciona el mecanismo. |
| Python for AI Development - aprenderemos Python mientras aprendemos solo lo relevante al campo | Python para Desarrollo de IA: uso de bibliotecas como: Pandas, NumPy y las bibliotecas SDK de modelos de lenguaje. | De escribir código a construir motores de datos: uso avanzado de bibliotecas de procesamiento de datos. Valor: La capacidad de transmitir información limpia y precisa a los modelos - la base de todo sistema de IA. |
| Using AI coding assistants - trabajaremos con asistentes de escritura Cursor o Copilot | Uso eficiente de asistentes de programación de IA: trabajar con herramientas como: Copilot o Cursor para mejorar la velocidad de desarrollo. | De escribir código a Code Review: cambiar el paradigma de trabajo de tus equipos. Valor: Gestionar Velocity mientras previenes "deuda técnica" y código que los desarrolladores no entienden completamente. |
| MLOps DevOps - conexión entre el mundo del software y el mundo de Operaciones, uso de contenedores | Gestión del Ciclo de Vida de ML: automatización de despliegue, monitoreo y actualización de modelos (similar a DevOps). | De experimento (PoC) a sistema de producción: la transición más difícil en el mundo de la IA. Valor: Construir un Pipeline automatizado que despliega y monitorea modelos sin que el sistema colapse bajo carga. |
| Frontier vs Open Source models - aprenderemos sobre modelos de OpenAI, Claude, Gemini y más, también habrá trabajo con herramientas Open Source | Modelos Cerrados (Frontier) vs. Open Source: comparación entre GPT/Gemini y modelos como Llama. | De dependencia de proveedores a independencia estratégica: entender cuándo "alquilar" un modelo (OpenAI) y cuándo "construir" (Llama). Valor: Proteger la privacidad del cliente y ahorrar costos ejecutando modelos locales personalizados. |
| RAG (Retrieval Augmentation Generation) - aprenderemos cómo mejoro los resultados de mi consulta contra datos que elijo | Generación Aumentada por Recuperación (RAG): "conectar" el modelo a datos organizacionales para prevenir alucinaciones. | De información general a información organizacional específica: prevenir "alucinaciones" del modelo basándose en hechos. Valor: Desafío arquitectónico de recuperar la información más precisa de bases de datos enormes para producir respuestas verídicas. |
| MCP (Model Context Protocol) - aprenderemos cómo permitir que la IA hable con sistemas externos | Protocolo de Contexto del Modelo: un nuevo estándar (de Anthropic) que permite a los modelos acceso uniforme a datos. | La revolución silenciosa en conectividad de datos: estandarización del Contexto. Valor: Conectar el modelo a APIs y tus datos de forma segura sin escribir "código pegamento" para cada modelo. |
| AI Agents & Agentic Design - aprenderemos cómo construir AI Agents correctamente y eficientemente, enseñando cómo conectarlos al mundo real | Agentes de IA y Patrones de Diseño Agéntico: transición de un modelo que "responde" a un modelo que "actúa" (planifica y ejecuta tareas). | De la etapa de "hacer preguntas" a la etapa de "ejecutar tareas": Valor: Construir un sistema autónomo que actúa (por ejemplo: un agente que encuentra una falla en el centro de llamadas, abre un ticket en Jira y actualiza al cliente). |
| LangChain, LangGraph, CrewAI - discutiremos bibliotecas que nos ayudan a construir AI Agents, también discutiremos cómo construirlos en Python Puro | Frameworks de IA: bibliotecas de Python para construir aplicaciones LLM y gestión de Estado. | De conversación simple a aplicación compleja: usar las herramientas correctas para gestión de lógica. Valor: Capacidad de gestionar Workflows complejos y mantener "Estado" a lo largo del tiempo. |
| Multi-Agent Orchestration - aprenderemos cómo trabajar con Multi-Agent y entenderemos por qué los necesitamos | Orquestación de Sistemas Multi-Agente: gestionar un equipo de agentes de IA trabajando juntos (uno programa, uno prueba). | Gestión de equipo virtual: dividir una tarea grande en subtareas enfocadas. Valor: Orquestar diferentes agentes (uno para transcripción, uno para análisis) para crear un producto final de calidad y controlado. |
| Performance and Scalability - también cubriremos este tema | Rendimiento y Escalabilidad: cómo hacer que un sistema de IA funcione rápido y sirva a miles de usuarios. | De sistema lento a sistema Enterprise: manejar cuellos de botella de IA. Valor: Aprenderás cómo gestionar tiempos de respuesta (Latency) y costos de tokens en sistemas que sirven a miles de usuarios simultáneamente. |
| Advanced AI Engineering - gestionar el Context es una de las cosas más importantes en la industria | Ingeniería de IA Avanzada: especialización en Context Engineering (construir el contexto perfecto para el modelo). | La evolución de Prompt Engineering: control total sobre el Input del modelo. Valor: Entregar la información más relevante para lograr resultados máximos a costo mínimo. |
| Monitoring and Logging - también cubriremos este tema | Monitoreo y Logging: seguimiento de la "salud" del modelo, costos (Tokens) y precisión de respuestas. | De suposiciones a diagnósticos: la capacidad de reproducir "¿por qué la máquina respondió así?". Valor: Construir un sistema transparente que permite Debugging y control sobre costos y calidad de productos en tiempo real. |
| Real life AI applications - desarrollaremos aplicaciones que se pueden usar en el mundo real y este también será el proyecto principal del curso, realmente construiremos un AI Agent que es un Customer support Agent | Aplicaciones del Mundo Real: casos de estudio de implementación de IA en productos existentes. | La línea de fondo: cómo convertir toda esta tecnología en valor empresarial (ROI) en productos como los que estás desarrollando. Valor: Aplicar todo el conocimiento técnico para un producto final que resuelve problemas reales para los usuarios. |
